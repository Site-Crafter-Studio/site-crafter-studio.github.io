Download the starter files


1. Go to this repository with the template files and click the green Code button.
2. Click Download ZIP and drag it into your desktop for now.



3. Double click on templates-main.zip to uncompress the folder on your desktop.
4. Open the uncompressed templates-main folder and make sure it contains the correct 4 folders: index, pages, projects, and work.


Create a GitHub account and a new repository


If you already have a GitHub account, skip to step #4.

1. Go to github.com and click Sign Up.
2. Enter your email, create a strong password, and enter a username.

⚠ Consider using your personal email, rather than a University-given email that will eventually expire.

⚠ The username you choose will appear in your website’s URL.

3. Complete account verification steps.
4. Click Create repository.

ⓘThe repository will store all your project’s files and each file’s revision history.

5. In the repository name field, enter [yourusername].github.io, where you replace [yourusername] with the username you chose during account setup. For example:



6. Ensure the privacy settings are set to Public.
7. In the “Initialize this repository with:” section, click Add a README file.
8. Click Create repository.
9. Click Add file and Upload file.



10. Drag and drop the 4 folders inside of the templates-main folder on your desktop.

⚠ Do not drag and drop the entire templates-main folder, but rather the 4 folders it contains. There should be a total of 16 files that are uploaded.

11. Scroll to the bottom, add a commit message, like “blank templates,” and click Commit changes.



Install Visual Studio Code

1. Go to Download Visual Studio Code and install the correct version of Visual Studio Code (VSCode) for your system.
2. Open the VSCode application.


Clone the repository

1. Go back to your repository’s home page in GitHub.
2. Click the green Code button and copy the HTTPS address to your clipboard.
3. In the Visual Studio Code application, create and open a new terminal.

ⓘ Terminal > New Terminal or use the command palette with cmd + shift + P and type/select the option Terminal: Create New Terminal.

4. Enter cd Desktop and hit return.
5. Enter git clone, paste the HTTPS address from GitHub, and hit return.

ⓘ git clone https://github.com/{your-username}/{your-username}.github.io.git


6. Authenticate to your GitHub account with your username and password. VSCode may open a window in your browser to further authorize your account.

ⓘ Now your GitHub repository is stored locally on your machine on your desktop and all changes will be saved to this location. You should see a new folder in your Desktop called {your-username}.github.io.

7. Go back to Visual Studio Code and click File > Open. Select the new folder (the clone of your repository) on your desktop.

ⓘ Now you should see the 4 folders (index, pages, projects, and work) and a README.md file in the Explorer menu on the left-hand side of the screen. The right-hand side of the screen is where you will be editing the files.


Push an update to GitHub

1. Open the README.md file from the Explorer menu on the left-hand side of the screen.
2. Replace the text with the following:
[My Website](https://{your-username}.github.io/pages/index.html)

ⓘ Your README file is written in Markdown. The Markdown syntax for adding a link is [text](hyperlink).

3. Save the file by clicking File > Save.
4. Click on the Source Control widget on the left-hand side of the screen. It will show that the README.md file has been changed.
5. Add a commit message like “update readme,” and click the blue Commit button.



6. Push the changes to your GitHub repository by clicking the blue ↑ button. Don’t worry about adding a message here.



7. Click OK, Don’t Show Again if a popup window, like this, shows up:



8. Go back to your repository’s home page on GitHub. You may need to refresh the page.
9. Your repository has been updated with the most recent changes you committed from Visual Studio Code. You can check the status of your latest commit and view all of your past commits here.



10. Click on the blue "My Website" link in the README section.

11. If you see this page, congratulations! You have successfully completed the Set Up steps. Switch between the About, Projects, and Work Experience tabs in the top navigation bar to view the different pages. This is what your website looks like with no customization or stylization (pretty boring, right?). Now let’s move on to the fun stuff!



12. Start customizing./a>
